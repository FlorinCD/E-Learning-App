Depth first search

Depth-First Search (DFS) is an algorithm used to traverse or search through a graph or tree data structure. It explores as far as possible along each branch before backtracking. DFS can be implemented using recursion or a stack data structure. It's often used to find paths, connected components, and cycles in a graph.

Here's how DFS works

  • Start at an initial node or vertex in the graph. This is the root of your search.
  • Mark the node as visited to avoid revisiting it.
  • Explore each unvisited adjacent node or neighbor of the current node. To do this, you can use a recursive function or a stack.
  • If you encounter an unvisited neighbor, visit it and repeat the process (recursively or by using a stack). This means you continue down the branch as far as possible before backtracking.
  • When you reach a node with no unvisited neighbors, backtrack to the previous node and continue the process until you have visited all nodes in the graph.
  • In this example, the dfs function starts at node 'A' and explores its neighbors, marking visited nodes and backtracking when there are no unvisited neighbors. The order in which nodes are visited depends on the specific structure of the graph and the starting node.

Usage of DFS

DFS is used in various applications, including finding connected components in a graph, determining if a path or cycle exists, and solving maze problems, among others. It's a fundamental algorithm in graph theory and computer science.

    Time complexity

    • In the worst case, DFS may visit every vertex and edge, exploring as deeply as possible along each branch before backtracking.
    • The time complexity of DFS is usually expressed as O(V + E), where V is the number of vertices and E is the number of edges in the graph.
    • Each vertex is visited once, and each edge is examined once.

    Space Complexity (Memory)

    • The space complexity is determined by the data structures used to implement the algorithm.
    • In DFS, a stack is often used to keep track of the vertices to be explored.
    • The space complexity of DFS is O(V) for the stack data structure due to the recursion or explicit stack usage.
    • In the case of recursion, the maximum depth of the recursion stack is the height of the recursion tree, which is the length of the longest path in the graph.

    It's important to note that the space complexity of DFS can be affected by the use of an additional data structure to keep track of visited vertices, which is often implemented using a boolean array or a hash set. This would add an additional O(V) space to the overall complexity.

    In summary:

    • Time Complexity: O(V + E)
    • Space Complexity: O(V)
