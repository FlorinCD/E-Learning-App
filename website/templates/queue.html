Queue

A queue is a fundamental data structure in computer science that models a collection of elements in a particular order. The defining feature of a queue is that it operates on a First-In-First-Out (FIFO) principle. This means that the first element added to the queue will be the first one to be removed, much like people standing in line to buy tickets at a movie theater: the person who arrives first is served first.

Queue Visualization

Characteristics of a Queue

FIFO Principle: The essential characteristic of a queue is that it follows the FIFO order. The first element enqueued (inserted) is the first one to be dequeued (removed).

Operations

  • Enqueue: Adding an element to the end of the queue.
  • Dequeue: Removing the element from the front of the queue.
  • Peek/Front: Viewing the element at the front without removing it.
  • IsEmpty: Checking if the queue is empty.
  • Size: Getting the number of elements in the queue.

Types of Queues

  • Simple Queue: Also known as a linear queue. Elements are added at the end and removed from the front. When it becomes full, no more elements can be added without dequeuing some first.
  • Circular Queue: In a circular queue, the last position is connected back to the first position to make a circle. This allows efficient use of storage, making it possible to use all available slots without needing to shift elements.
  • Priority Queue: Elements are processed based on their priority rather than their order in the queue. Higher priority elements are dequeued before lower priority ones. It can be implemented using a heap data structure for efficiency.
  • Deque (Double-ended Queue): A deque allows elements to be added or removed from both ends, providing greater flexibility than a simple queue.

Implementations of Queues

  • Array-Based Queue: An array is used to store the queue elements. Two pointers (or indices) track the front and rear of the queue. When the end of the array is reached, elements may need to be shifted or the array can be used in a circular manner.
  • Linked List-Based Queue: A linked list is used to store the queue elements. A node structure contains a value and a reference to the next node. The front and rear pointers track the beginning and end of the queue. This implementation allows dynamic resizing and avoids the need for shifting elements.

Applications of Queues

  • Scheduling Tasks: Operating systems use queues for job scheduling and managing tasks such as print jobs, process scheduling, and handling requests in servers.
  • Buffer Management: Queues are used in buffering data streams, such as in input/output (I/O) operations, where data needs to be managed between processes operating at different speeds.
  • Breadth-First Search (BFS): In graph traversal algorithms like BFS, a queue is used to keep track of the vertices to be explored, ensuring that each level of the graph is processed in order.
  • Simulation: Queues model real-world processes such as customers waiting in line, packets in networking, or requests to a web server.
  • Data Stream Processing: Queues handle real-time data streams where data arrives at a rapid rate, and needs to be processed in the order it arrives.
