Binary Search

Binary search is an efficient algorithm for finding an element in a sorted collection (array or list) by repeatedly dividing the search interval in half.

1. Initialization

  • Start with the entire collection.
  • Set two pointers, low and high, to the first and last elements of the collection, respectively.

2. Search Process

  • Calculate the middle index as mid = (low + high) / 2.
  • Compare the element at the middle index with the target element.
  • If the middle element is equal to the target, the search is successful, and the index is returned.
  • If the middle element is greater than the target, update high to mid - 1 and repeat the search in the left half.
  • If the middle element is less than the target, update low to mid + 1 and repeat the search in the right half.

3. Termination

  • Repeat the process until the low pointer is greater than the high pointer.
  • If the element is not found, return a special value indicating the absence of the target in the collection.

Usage of Binary Search

Binary search, known for its O(log n) time complexity, is widely applied across computer science and diverse real-world scenarios. Its efficiency in swiftly locating elements in sorted data finds applications in databases, file systems, spell-checking algorithms, game development, network routing, genetics, economics, telecommunications, online shopping, robotics, and medical imaging. The algorithm's ability to optimize search operations makes it a crucial tool in a broad spectrum of fields.

    Time complexity

    • Binary search has a time complexity of O(log n), where n is the number of elements in the collection. This makes it significantly faster than linear search algorithms, especially for large datasets. However, it's important to note that binary search can only be applied to sorted collections. If the collection is not sorted, a preprocessing step to sort it is required, which could affect the overall efficiency.

    Space Complexity (Memory)

    • Binary search typically uses a constant amount of additional memory space, also known as O(1) space complexity. The space complexity of an algorithm refers to the amount of memory it uses relative to the input size.
